# Data Structures and Complexity Basics

## What it is
Core data structures (arrays, hash maps, trees, graphs) and time/space
complexity for reasoning about performance.

## Why it matters
You need to select data structures that keep pipelines responsive and scalable.

## Core ideas
- Big-O time and space
- Arrays vs hash maps vs trees
- Graph representations and traversal cost

## Example
A hash map gives expected O(1) lookup for feature dictionaries; a sorted list
gives O(log n) with binary search but O(n) insertions.

## Pitfalls
- Ignoring worst-case behavior for hash maps
- Using dense matrices for sparse data
- Underestimating memory usage of nested structures
