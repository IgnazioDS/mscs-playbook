# Docs

This directory is the lightweight documentation hub for the repository.

It does not currently act as a second full documentation system. Its job is to point readers toward the primary navigation surfaces that already exist elsewhere in the repo.

## What This Section Is For

- orienting a new reader to the repository structure
- linking the curriculum layer to the project layer
- keeping repo-level navigation separate from module-level content

## Current Navigation Sources

- [Root README](../README.md): top-level repository positioning and current value
- [Repository Structure](../STRUCTURE.md): concise structure reference
- [Modules Index](../modules/README.md): curriculum overview across modules `00` through `17`
- [Projects Index](../projects/README.md): portfolio-oriented project overview

## Current State

The repository's strongest navigation surfaces today are the root README and the module READMEs themselves. The `docs/` layer is intentionally light and should stay factual about what already exists in the repo.

## Recommended Reading Flow

1. Start with the [Root README](../README.md).
2. Use the [Modules Index](../modules/README.md) to pick a curriculum area.
3. Enter a module and follow its numbered concept pages in order.
4. Use the [Projects Index](../projects/README.md) when you want applied, portfolio-facing work tied back to the modules.
